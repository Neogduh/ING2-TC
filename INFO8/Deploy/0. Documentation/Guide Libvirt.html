Guide Libvirt

Libvirt sur le PIE

Les TP 2 à 4 demandent de travailler dans des machines virtuelles. Si vous utilisez les postes de salles machines, il y a certaines particularités à prendre en compte. L’outil sur lequel vous devez travailler est libvirt via son interface virt-manager.

En cas de difficulté pour suivre ce guide, une vidéo démonstration est disponible à cette adresse.

Premier démarrage

Ouvrez virt-manager. Au premier démarrage, vous devez ajouter une connexion. Allez dans File puis Add connection. Sélectionnez QEMU/KVM user session et cliquez sur Connect.

Faites clic droit sur QEMU/KVM User session et allez dans Details. Dans l’onglet Storage, cliquez sur le + pour ajouter une pool de stockage. Ajoutez une pool de stockage appelée pool de type dir qui pointe sur /srv/libvirt-workdir.

⚠️ Attention

Vous devez impérativement utiliser ce dossier pour stocker les disques de vos VM et les fichiers iso d’installation.

DANGER

Ce dossier est vidé à chaque reboot, ne stockez rien d’important dedans.

Les raisons pour lesquelles ce fonctionnement est nécessaire sont que vous ne disposez pas d’assez de place dans votre AFS pour stocker des VM et c’est de toute façon trop lent. Si vous utilisez un autre dossier que celui précisé au dessus, le stockage de votre VM sera stockée en RAM ce qui posera des problèmes de place.

Création d’une VM

Munissez vous de l’ISO d’Ubuntu Server 22.04

Pour limiter la charge sur les serveurs d’Ubuntu, utilisez ce torrent pour télécharger le fichier.

Pour l’utiliser, utilisez les commandes suivantes :

cd /srv/libvirt-workdir/
aria2c --enable-dht=false --enable-dht6=false --seed-ratio=0 ~/Downloads/ubuntu-22.04.3-live-server-amd64.iso.torrent

Lorsque le fichier est téléchargé, merci de laisser tourner aria2c pour partager le fichier avec les autres ordinateurs et limiter la charge sur le réseau de la salle serveur de l’école. Merci ! 🙏

Faites clic droit sur QEMU/KVM User session puis New.

Sélectionnez Local install media (ISO image or CDROM)

Sélectionnez le fichier ISO téléchargé. Normalement, il devrait être dans la pool de stockage pool (donc dans /srv/libvirt-workdir). Décochez Automatically detect from the installation media / source et sélectionnez Ubuntu 22.04 LTS.

À l’étape de création du disque, sélectionnez Select or create custom storage. Dans Manage, sélectionnez pool et ajoutez un nouveau volume.

Enfin, avant de finaliser l’installation de la VM, dans Network selection sélectionnez Bridge device... et entrez virbr0.
